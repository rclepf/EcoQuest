EcoQuest: A Trilha Sustentável

Um guia laboratorial passo a passo completo, hiper detalhado, focado nos fundamentos de GameObjects, Components, Física Rígida (Rigidbody), Triggers estruturais, padrões estruturais Singleton e ciclos de execução C#.

1. Visão Geral & Objetivos de Aprendizagem

A proposta do EcoQuest é converter conceitos de física mecânica abstrata e lógica sequencial estruturada em mecânicas interativas de jogo. Este projeto adapta a arquitetura do clássico "Cubo Explorador" em uma dinâmica de jogo "Endless Runner/Trilha Ativa".

O jogador controla o Eco-Robô, que avança de maneira retilínea uniforme por uma trilha flutuante. O desafio consiste em executar desvios laterais mecânicos, coletar detritos recicláveis para pontuar, disparar sementes purificadoras contra barreiras industriais tóxicas e alcançar o Santuário Verde intacto.

O que você aprenderá neste módulo avançado:

Manipulação de forças físicas no FixedUpdate, controle estrutural de ciclo de vida de instâncias dinâmicas via Instantiate e Destroy, tratamento seguro de colisões não-físicas por zonas de gatilho (Triggers) e centralização de estados voláteis de dados com o padrão Singleton.

3. Matriz Arquitetural de Tags

Tags funcionam como assinaturas de strings estáticas otimizadas em nível de engine para triagem de metadados durante eventos de contato de colisores. A omissão ou digitação incorreta destas chaves anulará as validações condicionais do código C#.

Atenção Crucial ao "Case Sensitivity"

A engine trata Reciclavio e reciclavio como sequências distintas em memória. Siga a tabela abaixo estritamente sem o uso de caracteres especiais ou acentuações idiomáticas.

Identificador Exato (Tag) Alvo de Atribuição na Hierarquia Finalidade Operacional no Código
Player GameObject do Eco-Robô Garante que colisores externos identifiquem a entidade do usuário.
Reciclavio Prefabs de detritos espalhados (Cilindros) Informa ao gatilho que a pontuação global deve ser incrementada.
Poluicao Obstáculos dinâmicos de fumaça (Cubos) Desencadeia a rotina de subtração de vidas do jogador.
Semente Instâncias de projéteis ativos (Esferas) Valida o vetor de ataque contra os bloqueios tóxicos da pista.
BarreiraToxica Estruturas estáticas de contenção (Barris) Alvo exclusivo destrutível pelo impacto direto do projétil Semente.
Santuario Portal místico final de fase Sinaliza ao GameManager a interrupção do loop de gameplay e tela de vitória.

Como Configurar: Acesse Edit > Project Settings > Tags and Layers. Clique no símbolo de soma + sob a aba Tags e insira individualmente os registros acima.

4. Missão 1: O Eco-Robô (Física Dinâmica e Movimentação)

Nesta etapa, criaremos o corpo físico do nosso herói tecnológico e daremos a ele a capacidade de locomoção contínua e desvio paramétrico responsivo.

  • 1

    Construção Estrutural da Pista

    No painel Hierarchy, execute Create > 3D Object > Cube. Renomeie o elemento para Trilha_Principal. No Inspector, altere o componente Transform para os seguintes valores absolutos: Position (0, 0, 0) e Scale (12, 1, 100).

  • 2

    Instanciação do Jogador

    Crie um novo objeto Cube na raiz da cena, batize-o de EcoRobo e mude sua coordenada de inicialização para Position (0, 1, -45). Defina a Tag do objeto como Player.

  • 3

    Atribuição Física (Rigidbody)

    Com o EcoRobo selecionado, clique em Add Component e busque por Rigidbody. Expanda o menu de propriedades Constraints e marque obrigatoriamente as flags Freeze Rotation X e Freeze Rotation Z. Isso impede que forças de fricção façam o robô capotar lateralmente.

PlayerController.cs C# / Unity Standard 
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [Header("Configurações de Vetor")]
    public float forwardSpeed = 8f;
    public float lateralSpeed = 12f;
    public float trackLimit = 5.5f;

    private Rigidbody rb;
    private float sideInput;

    void Start()
    {
        rb = GetComponent<Rigidbody>();
        if (rb == null) 
            Debug.LogError("ERRO FATAL: Rigidbody ausente em EcoRobo!");
    }

    void Update()
    {
        // Captura de input discreto e sem suavização para respostas instantâneas de arcade
        sideInput = Input.GetAxisRaw("Horizontal");
    }

    void FixedUpdate()
    {
        // Cálculo vetorial linear para avanço cinemático contínuo
        Vector3 forwardMove = transform.forward * forwardSpeed * Time.fixedDeltaTime;
        Vector3 lateralMove = transform.right * sideInput * lateralSpeed * Time.fixedDeltaTime;
        
        Vector3 finalTargetPosition = rb.position + forwardMove + lateralMove;

        // Aplicação de restrição matemática limitadora de borda (Clamp)
        finalTargetPosition.x = Mathf.Clamp(finalTargetPosition.x, -trackLimit, trackLimit);

        // Movimentação física segura por interpolação de matriz de colisão
        rb.MovePosition(finalTargetPosition);
    }
}

5. Missão 2: Câmera Cinematográfica Suave (Lerp)

Câmeras fixas geram desconforto visual em jogos rápidos. Usaremos Interpolação Linear de Vetores (Lerp) acoplada ao ciclo tardio de atualização (LateUpdate) para alcançar estabilidade visual de nível profissional.

Por que LateUpdate?

Se atualizarmos a posição da câmera no método Update ou FixedUpdate, ocorrerá um fenômeno de oscilação e trepidação ("jittering"), pois a câmera tentará se mover antes ou concorrentemente à física do Robô. O LateUpdate garante execução sequencial controlada posterior.

CameraFollow.cs C# / Unity Standard 
using UnityEngine;

public class CameraFollow : MonoBehaviour
{
    public Transform targetPlayer;
    public Vector3 offsetDistance = new Vector3(0f, 4.5f, -8f);
    public float smoothFactor = 5f;

    void LateUpdate()
    {
        if (targetPlayer == null) return;

        // Projeta o ponto de destino ideal no espaço euclidiano 3D
        Vector3 targetDestination = targetPlayer.position + offsetDistance;
        
        // Interpola suavemente a posição atual e o destino em função do delta de tempo
        Vector3 smoothPosition = Vector3.Lerp(transform.position, targetDestination, smoothFactor * Time.deltaTime);
        
        transform.position = smoothPosition;

        // Força o enquadramento angular da câmera para olhar fixamente para as coordenadas do Robô
        transform.LookAt(targetPlayer.position + Vector3.up * 0.5f);
    }
}

Passo operacional: Anexe o script à Main Camera da cena. No Inspector, arraste o objeto EcoRobo da hierarquia para o campo de referência vazio chamado Target Player.

6. Missão 3: Coletáveis Sustentáveis e o Padrão Singleton

Para contabilizar os pontos do jogador de forma centralizada e acessível por qualquer script da cena, criaremos um GameManager estruturado sob o padrão de projeto Singleton.

GameManager.cs C# / Core Singleton 
using UnityEngine;

public class GameManager : MonoBehaviour
{
    // Instância global de acesso público estático
    public static GameManager Instance { get; private set; }

    [Header("Indicadores de Estado")]
    public int totalScore = 0;
    public int activeVidas = 3;
    public bool isLevelComplete = false;

    void Awake()
    {
        // Garantia irrestrita do ciclo de instância única na memória da aplicação
        if (Instance == null)
        {
            Instance = this;
        }
        else
        {
            Destroy(gameObject);
        }
    }

    public void AddPoints(int pointsValue)
    {
        totalScore += pointsValue;
        Debug.Log("Pontuação Atualizada: " + totalScore + " Pontos.");
    }

    public void ApplyDamage()
    {
        activeVidas--;
        Debug.LogWarning("Dano Registrado! Restam: " + activeVidas + " vidas.");
        if (activeVidas <= 0)
        {
            Debug.LogError("FIM DE JOGO - O robô colapsou energeticamente.");
            // Lógica de reinicialização futura
        }
    }
}

Agora, implementaremos a lógica individual de colisão não-física dos resíduos recicláveis usando a sub-rotina nativa OnTriggerEnter.

ColetavelItem.cs C# / Trigger Matrix 
using UnityEngine;

public class ColetavelItem : MonoBehaviour
{
    public int scoreReward = 15;
    public float rotateSpeed = 120f;

    void Update()
    {
        // Criação do feedback visual rotatório e constante de atração técnica
        transform.Rotate(Vector3.up * rotateSpeed * Time.deltaTime, Space.World);
    }

    void OnTriggerEnter(Collider other)
    {
        // Filtra e valida se o elemento intruso possui o rótulo Player atribuído
        if (other.CompareTag("Player"))
        {
            // Executa a comunicação segura via canal Singleton do GameManager
            GameManager.Instance.AddPoints(scoreReward);
            
            // Libera o endereço de memória destruindo a instância local do item
            Destroy(gameObject);
        }
    }
}

Workflow de Geração do Prefab Coletável

1. Crie um Cylinder na cena, mude o nome para Item_Plastico e mude a Tag para Reciclavio.
2. No componente Capsule Collider do inspetor, marque obrigatoriamente a opção Is Trigger.
3. Anexe o script ColetavelItem.cs a ele.
4. Arraste este cilindro da hierarquia diretamente para sua pasta Assets/_Project/Prefabs/. Delete o objeto original da cena. Agora ele é um modelo de clonagem global reutilizável (Prefab)!

7. Missão 4: Chuva de Poluição (Spawn Dinâmico Aleatório)

Para criar um desafio dinâmico, criaremos um algoritmo responsável por monitorar o tempo de jogo e instanciar obstáculos (nuvens de poluição) em localizações horizontais calculadas de forma randômica à frente do avanço do jogador.

ObstaculoFumaca.cs C# / Obstacle Movement 
using UnityEngine;

public class ObstaculoFumaca : MonoBehaviour
{
    public float expansionSpeed = 2f;
    
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            GameManager.Instance.ApplyDamage();
            Destroy(gameObject);
        }
    }
}
GeradorObstaculos.cs C# / Dynamic Factory 
using UnityEngine;

public class GeradorObstaculos : MonoBehaviour
{
    public GameObject prefabPoluicao;
    public Transform playerTransform;
    
    public float spawnInterval = 1.8f;
    public float spawnDistanceAhead = 40f;
    public float xSpawnRange = 5f;

    private float timerAccumulator = 0f;

    void Update()
    {
        if (playerTransform == null) return;

        timerAccumulator += Time.deltaTime;

        if (timerAccumulator >= spawnInterval)
        {
            SpawnObstaculoAleatorio();
            timerAccumulator = 0f;
        }
    }

    void SpawnObstaculoAleatorio()
    {
        // Computa uma coordenada X pseudo-aleatória dentro dos limites de largura da pista
        float randomX = Random.Range(-xSpawnRange, xSpawnRange);
        
        // Centraliza o vetor de spawn fixando uma distância segura à frente do robô
        Vector3 spawnPosition = new Vector3(randomX, 1f, playerTransform.position.z + spawnDistanceAhead);

        // Instanciação em tempo de execução nativa
        Instantiate(prefabPoluicao, spawnPosition, Quaternion.identity);
    }
}

Passo Operacional: Salve o objeto de fumaça cinza como Prefab (com colisor configurado em Is Trigger e a tag Poluicao). Crie um objeto vazio chamado Mecanismo_Spawner na cena e configure as referências correspondentes no script.

8. Missão 5: Disparo de Sementes Purificadoras e Destruição

Para contornar o problema dos barris industriais intransponíveis na pista, o jogador ganhará o poder de lançar projéteis cinemáticos capazes de limpar e fragmentar objetos destrutíveis.

ProjetilSemente.cs C# / Physics Matrix 
using UnityEngine;

public class ProjetilSemente : MonoBehaviour
{
    public float projectileSpeed = 25f;
    public float lifeTime = 3f;

    void Start()
    {
        // Evita o vazamento crônico de memória na pilha destruindo o projétil por timeout
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        // Deslocamento cinemático espacial retilíneo acelerado
        transform.Translate(Vector3.forward * projectileSpeed * Time.deltaTime);
    }

    void OnTriggerEnter(Collider other)
    {
        // Se colidir contra um elemento rotulado como barreira impenetrável
        if (other.CompareTag("BarreiraToxica"))
        {
            Destroy(other.gameObject); // Elimina o obstáculo da cena
            Destroy(gameObject);       // Elimina a si mesmo
        }
    }
}

Acoplamento no PlayerController

Abra seu arquivo PlayerController.cs criado na Missão 1 e adicione duas variáveis globais públicas: public GameObject sementePrefab; e public Transform spawnPoint;. Dentro do escopo da função Update(), insira a seguinte condicional estrita:

if (Input.GetKeyDown(KeyCode.Space)) 
{
    Instantiate(sementePrefab, spawnPoint.position, Quaternion.identity);
}

9. Missão 6: O Portal do Santuário e Fim de Jogo

Toda experiência necessita de uma condição de término bem estabelecida. O Santuário Verde atuará como o ponto geográfico final da trilha.

SantuarioPortal.cs C# / Win Condition 
using UnityEngine;

public class SantuarioPortal : MonoBehaviour
{
    void OnTriggerEnter(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            GameManager.Instance.isLevelComplete = true;
            Debug.Log("VITÓRIA SUPREMA: A floresta foi restaurada pelo EcoQuest!");
            
            // Congela o fluxo temporal do motor físico da Unity
            Time.timeScale = 0f;
        }
    }
}

Montagem do Cenário Final: Na extremidade oposta da pista (ex: posição Z = 45), instancie um cilindro ou portal decorativo translúcido de grande escala. Configure a Tag para Santuario, ative o parâmetro Is Trigger em seu colisor e monte o script associado.
