GUIA DE DESENVOLVIMENTO DE JOGO

EcoQuest: A Trilha Sustentável

Manual laboratorial exaustivo de desenvolvimento na Unity Engine. Todas as instruções foram expandidas com explicações detalhadas linha a linha e caminhos de interface destacados com máximo contraste visual, eliminando a necessidade de seleção de texto para leitura.

1. Visão Geral e Objetivos do Projeto

O EcoQuest é uma Atividade Prática Guiada projetada para consolidar os fundamentos mecânicos da Unity Engine através da criação de um mini-jogo no estilo Endless Runner focado em sustentabilidade ambiental.

A dinâmica de jogo é direta e focada: o jogador assume o papel do Eco-Robô, um autômato cilíndrico ou cúbico que se desloca de forma contínua e retilínea por uma pista suspensa no espaço. O percurso está repleto de desafios ecológicos. Para sobreviver e triunfar, o aluno deve programar sistemas para desviar de barreiras de fumaça poluente, coletar detritos recicláveis para acumular pontuação, disparar sementes purificadoras para quebrar obstáculos físicos e cruzar o portal místico do Santuário Verde.

O que será avaliado técnica e conceitualmente nesta atividade:

• Gerenciamento dinâmico de forças através do motor físico integrado (componente Rigidbody).
• Ciclo de vida de objetos com instanciação programática (Instantiate) e coleta de lixo de memória (Destroy).
• Detecção precisa de colisões não-físicas por sobreposição de volumes (mecanismo de Triggers).
• Centralização lógica e arquitetura limpa de dados voláteis usando o padrão estrutural Singleton.

3. Configuração do Sistema Condicional de Tags

As Tags funcionam como rótulos identificadores baseados em texto. Quando um objeto bate em outro, o script lê essa string para decidir se aplica dano, soma pontos ou reinicia o jogo. Veja abaixo a matriz exata de tags que você deve registrar no sistema interno da Unity.

Atenção Ortográfica Estrita (Case Sensitive)

O computador diferencia letras maiúsculas de minúsculas. Se você escrever Semente no gerenciador de Tags e no código digitar semente, o jogo apresentará uma falha lógica silenciosa e o projétil atravessará a barreira sem quebrá-la. Digite com atenção.

Texto Exato da Tag Destinatário da Configuração na Hierarquia Explicação da Utilidade Prática
Player Objeto do Eco-Robô (Jogador) Informa ao spawner e itens de coleta que a entidade que colidiu é o usuário.
Reciclavio Modelos de lixo (Garrafas/Cilindros) Informa ao gatilho que a rotina de adição de pontos deve ser executada.
Poluicao Obstáculos de fumaça cinza (Cubos) Dispara a chamada do método que reduz o contador de vidas do jogador.
Semente Esferas de energia disparadas (Projéteis) Diz às barreiras fixas que elas foram atingidas pelo ataque purificador.
BarreiraToxica Barris industriais estáticos de bloqueio Define que o objeto pode ser quebrado quando atingido por uma Semente.
Santuario O grande portal verde no fim do mapa Ativa o encerramento do nível e exibe o log de vitória na tela.
Como Criar e Registrar na Unity:

Clique em qualquer objeto na aba Hierarchy. Olhe para a aba direita Inspector, localize o campo superior escrito Tag (que por padrão está como *Untagged*), clique nele e selecione Add Tag.... Na janela que se abre, clique no botão de mais +, digite o nome listado na tabela acima e clique em Save. Repita o processo para as seis tags.

4. Missão 1: O Eco-Robô (Física Cinemática e Movimento do Jogador)

Nesta missão, construiremos o chão do jogo, o personagem principal e configuraremos as forças físicas necessárias para que ele ande automaticamente para a frente enquanto responde a comandos de desvio laterais.

Passo 1: Construindo a Geometria do Cenário e do Jogador

1. Na aba Hierarchy, clique com o botão direito em um espaço vazio e escolha 3D Object > Cube. Renomeie o objeto imediatamente para Trilha_Principal. No painel Inspector (à direita), mude os valores do componente **Transform** para: **Position (0, 0, 0)** e **Scale (12, 1, 100)**. Isto criará uma longa rodovia flutuante de testes.

2. Na mesma aba Hierarchy, crie outro 3D Object > Cube. Renomeie-o para EcoRobo. Mude as configurações do componente **Transform** para: **Position (0, 1, -45)**. Isso posicionará o robô no início da pista, um pouco acima do chão para evitar colisões erradas. No topo do Inspector deste objeto, altere o campo Tag para Player.

Passo 2: Configurando as Restrições Físicas

Selecione o EcoRobo. No Inspector, clique no botão inferior Add Component, digite Rigidbody e aperte Enter. Com o componente adicionado, localize a propriedade chamada Constraints e clique na seta para expandir suas opções. Marque obrigatoriamente as caixas de seleção Freeze Rotation X e Freeze Rotation Z.

Por que ativamos o Freeze Rotation?

Ao andar para os lados rapidamente, o atrito físico com o chão da pista pode gerar forças de torque rotacional. Se essas rotações não forem travadas (congeladas), o robô tombará como um dado e começará a rolar descontroladamente pelo cenário, quebrando o controle do jogo.

Passo 3: Criando e Escrevendo o Script de Movimentação

Vá até a pasta Assets > _Project > Scripts na aba Project. Clique com o botão direito em um espaço vazio, escolha Create > C# Script e nomeie o arquivo exatamente como PlayerController. Dê um duplo clique para abrir no editor de código e cole a estrutura lógica detalhada abaixo:

PlayerController.cs C# / Unity Architecture 
using UnityEngine;

public class PlayerController : MonoBehaviour
{
    [Header("Configurações de Velocidade")]
    public float forwardSpeed = 8f;    // Velocidade constante de avanço automático
    public float lateralSpeed = 12f;   // Velocidade de resposta dos desvios laterais
    public float trackLimit = 5.5f;    // Limite lateral em metros para o robô não cair da pista

    private Rigidbody rb;              // Armazena a referência interna do componente físico
    private float sideInput;           // Armazena o valor do comando horizontal do jogador

    void Start()
    {
        // Busca automaticamente o Rigidbody atachado neste mesmo GameObject
        rb = GetComponent<Rigidbody>();
        
        // Verificação de segurança para evitar erros de ponteiro nulo
        if (rb == null) 
            Debug.LogError("ERRO: Rigidbody não foi encontrado no EcoRobo!");
    }

    void Update()
    {
        // Captura a entrada de teclado nas teclas A/D ou Setas Direcionais
        // GetAxisRaw retorna valores puros (-1, 0 ou 1) para respostas instantâneas
        sideInput = Input.GetAxisRaw("Horizontal");
    }

    void FixedUpdate()
    {
        // Calcula o vetor de movimento para frente baseado no tempo fixo da física
        Vector3 forwardMove = transform.forward * forwardSpeed * Time.fixedDeltaTime;
        
        // Calcula o vetor de movimento lateral baseado no comando do teclado
        Vector3 lateralMove = transform.right * sideInput * lateralSpeed * Time.fixedDeltaTime;
        
        // Soma a posição atual do Rigidbody com os vetores calculados de destino
        Vector3 finalTargetPosition = rb.position + forwardMove + lateralMove;

        // Restringe matematicamente o eixo X para que ele fique preso dentro dos limites da rodovia
        finalTargetPosition.x = Mathf.Clamp(finalTargetPosition.x, -trackLimit, trackLimit);

        // Move o corpo físico considerando colisões e barreiras físicas ao longo do caminho
        rb.MovePosition(finalTargetPosition);
    }
}

Vinculação Técnica Final: Arraste o arquivo PlayerController de dentro do painel Project e solte-o em cima do objeto EcoRobo na aba Hierarchy.

5. Missão 2: Câmera Suave (Interpolação Vetorial Linear - Lerp)

A câmera padrão da Unity é estática. Se o robô andar para frente, ele sairá do enquadramento do monitor. Criaremos um script que faz a câmera perseguir o robô mantendo um distanciamento confortável e amortecido.

Entendendo a Arquitetura do Script

O segredo de uma câmera fluida é usar o método Vector3.Lerp(A, B, t). Ele calcula uma posição intermediária fracionada entre a localização atual da câmera (ponto A) e a localização traseira ideal do robô (ponto B) com base em um fator de amortecimento (t). Isso cria um efeito elástico visual muito agradável.

CameraFollow.cs C# / Rendering Execution Order 
using UnityEngine;

public class CameraFollow : MonoBehaviour
{
    public Transform targetPlayer;       // Espaço no Inspector para arrastar o objeto do Eco-Robô
    public Vector3 offsetDistance = new Vector3(0f, 4.5f, -8f); // Distância de enquadramento (X=centralizado, Y=alto, Z=atrás)
    public float smoothFactor = 5f;       // Fator de atraso/suavização do movimento elástico

    void LateUpdate()
    {
        // Validação de salvaguarda: se esquecer de arrastar o jogador, o script cancela a execução para não travar o jogo
        if (targetPlayer == null) return;

        // Define matematicamente onde a câmera deveria estar no espaço 3D
        Vector3 targetDestination = targetPlayer.position + offsetDistance;
        
        // Executa a aproximação suave fracionada entre a posição atual e o destino ideal
        Vector3 smoothPosition = Vector3.Lerp(transform.position, targetDestination, smoothFactor * Time.deltaTime);
        
        // Atualiza a posição real da câmera com o vetor suavizado
        transform.position = smoothPosition;

        // Rotaciona e aponta as lentes da câmera diretamente para a base do robô, mantendo-o sempre no centro do monitor
        transform.LookAt(targetPlayer.position + Vector3.up * 0.5f);
    }
}

Por que usamos obrigatoriamente o método LateUpdate?

A Unity processa a movimentação física do jogador dentro do FixedUpdate. Se movêssemos a câmera dentro do Update normal, os dois métodos competiriam por prioridade na CPU de forma dessincronizada, fazendo com que a câmera desse pequenos pulos e tremesse na tela. O LateUpdate executa por último na fila de frames, garantindo que o jogador já se moveu e a câmera apenas o segue estavelmente.

Configurando a Câmera na Interface: Selecione o objeto Main Camera na aba Hierarchy. Arraste o arquivo CameraFollow para dentro do Inspector da câmera. Com o script anexado, clique e segure no objeto EcoRobo da Hierarchy, arraste-o com o mouse e solte-o dentro da caixinha vazia chamada Target Player no script da câmera.

6. Missão 3: Sistema de Coleta Sustentável e o Padrão Arquitetural Singleton

Nesta etapa criaremos o cérebro abstrato do jogo: um gerenciador de estados capaz de registrar o score e a quantidade de vidas. Usaremos o padrão de desenvolvimento Singleton para criar um canal global de comunicação direta entre os objetos da cena.

Entendendo o Padrão Singleton

Quando um item de lixo é coletado, ele precisa somar pontos em um placar geral. Sem o Singleton, cada garrafa na pista teria que fazer uma busca lenta na memória para descobrir onde o painel de pontuação está. O Singleton cria uma propriedade pública estática na memória que funciona como um interfone central de discagem instantânea de qualquer parte do código através da chamada simples de GameManager.Instance.

GameManager.cs C# / Central Architecture 
using UnityEngine;

public class GameManager : MonoBehaviour
{
    // Propriedade estática global que concede acesso externo seguro
    public static GameManager Instance { get; private set; }

    [Header("Estado Global de Jogo")]
    public int totalScore = 0;         // Armazena os pontos acumulados de reciclagem
    public int activeVidas = 3;        // Contador ativo de integridade do Robô
    public bool isLevelComplete = false; // Chave booleana de controle de fim de nível

    void Awake()
    {
        // Implementação rigorosa do padrão Singleton para evitar duplicidade de gerêncial e conflito de memória
        if (Instance == null)
        {
            Instance = this; // Registra esta instância como o canal oficial
        }
        else
        {
            Destroy(gameObject); // Se nascer outro GameManager duplicado por erro, destrói o impostor imediatamente
        }
    }

    public void AddPoints(int pointsValue)
    {
        totalScore += pointsValue;
        Debug.Log("Pontuação acrescida com sucesso! Total atual: " + totalScore + " pontos.");
    }

    public void ApplyDamage()
    {
        activeVidas--;
        Debug.LogWarning("Alerta de Colisão! Integridade reduzida. Restam: " + activeVidas + " unidades.");
        
        if (activeVidas <= 0)
        {
            Debug.LogError("CRÍTICO: O Eco-Robô sofreu colapso estrutural. Game Over!");
            Time.timeScale = 0f; // Congela o andamento do jogo
        }
    }
}

Ativação do Gerenciador: Crie um objeto vazio na hierarquia clicando com o botão direito e selecionando Create Empty. Nomeie o elemento exatamente como GameManager e associe o script GameManager a ele.

Passo 2: Programando a Lógica Física de Gatilho do Item Coletável

Crie um cilindro na cena (3D Object > Cylinder), nomeie como Item_Reciclavio e altere sua escala para **Scale (0.6, 0.6, 0.6)** para que ele tenha o tamanho adequado. No topo do Inspector, defina sua Tag como Reciclavio.

No componente **Capsule Collider** deste objeto, **marque obrigatoriamente a caixa de seleção Is Trigger**. Isso transforma a barreira física sólida em um campo de força de gatilho fantasma, permitindo que o robô atravesse o objeto e dispare o código abaixo:

ColetavelItem.cs C# / Collision Systems 
using UnityEngine;

public class ColetavelItem : MonoBehaviour
{
    public int scoreReward = 15;       // Quantidade de pontos que este item específico concede
    public float rotateSpeed = 120f;    // Velocidade angular de rotação do item na pista

    void Update()
    {
        // Faz o objeto girar constantemente no próprio eixo vertical para gerar apelo visual clássico de moedas
        transform.Rotate(Vector3.up * rotateSpeed * Time.deltaTime, Space.World);
    }

    void OnTriggerEnter(Collider other)
    {
        // Verifica se a entidade física que invadiu o campo de força possui a tag Player
        if (other.CompareTag("Player"))
        {
            // Disca via interfone Singleton e chama o método de somar pontos passando a recompensa
            GameManager.Instance.AddPoints(scoreReward);
            
            // Destrói a própria instância do item do cenário para ele sumir e liberar memória RAM
            Destroy(gameObject);
        }
    }
}
Passo 3: Salvando o Objeto como Prefab Corporativo

Com o script ColetavelItem anexado ao objeto Item_Reciclavio na cena e a flag *Is Trigger* ligada, clique sobre o objeto Item_Reciclavio na hierarquia, segure o clique e arraste-o para baixo, soltando-o dentro da pasta de arquivos Assets > _Project > Prefabs. O nome do objeto na hierarquia ficará **Azul escuro**, indicando que agora ele é um Prefab oficial pronto para ser gerado aos milhares por código! Delete o objeto original que sobrou na hierarquia da cena.

7. Missão 4: Chuva de Poluição (Fábrica Dinâmica de Obstáculos e Spawner Automático)

Jogos lineares perdem a graça rapidamente. Para dar dinamismo à gameplay, criaremos um sistema inteligente de controle temporal que cria obstáculos perigosos de forma aleatória na pista à frente do jogador.

Passo 1: O Objeto de Dano da Fumaça

Crie um 3D Object > Cube na hierarquia, batize de Nuvem_Poluicao, mude sua Tag para Poluicao, configure o colisor como **Is Trigger** e anexe o script ObstaculoFumaca abaixo. Salve o objeto na pasta Prefabs e apague o modelo da cena da mesma forma que fizemos com o coletável.

ObstaculoFumaca.cs C# / Hazard System 
using UnityEngine;

public class ObstaculoFumaca : MonoBehaviour
{
    void OnTriggerEnter(Collider other)
    {
        // Se colidir com o jogador, acessa o Singleton e aplica penalidade de vida
        if (other.CompareTag("Player"))
        {
            GameManager.Instance.ApplyDamage();
            Destroy(gameObject); // Some da pista para não causar múltiplos danos seguidos
        }
    }
}
Passo 2: O Motor Automatizado de Instanciação

Crie um objeto vazio chamado Mecanismo_Spawner. Escreva o código GeradorObstaculos e anexe a ele. Este código acumula o tempo de jogo e usa a função Instantiate para criar cópias físicas perfeitas do obstáculo em posições horizontais randômicas obtidas via Random.Range.

GeradorObstaculos.cs C# / Dynamic Factory Pattern 
using UnityEngine;

public class GeradorObstaculos : MonoBehaviour
{
    public GameObject prefabPoluicao;     // Gaveta no Inspector para receber o Prefab salvo em disco
    public Transform playerTransform;      // Referência de posicionamento do Robô
    
    public float spawnInterval = 1.8f;     // Intervalo temporal fixo em segundos entre cada nascimento
    public float spawnDistanceAhead = 40f; // Distância fixa de surgimento à frente das coordenadas do robô
    public float xSpawnRange = 5f;         // Largura máxima permitida para o cálculo aleatório horizontal

    private float timerAccumulator = 0f;   // Variável interna de controle de relógio

    void Update()
    {
        if (playerTransform == null) return;

        // Incrementa o cronômetro com base na fração de tempo decorrida do último frame físico
        timerAccumulator += Time.deltaTime;

        // Quando estourar o limite de tempo estipulado
        if (timerAccumulator >= spawnInterval)
        {
            SpawnObstaculoAleatorio();
            timerAccumulator = 0f;         // Zera o relógio interno para recomeçar o ciclo de loop
        }
    }

    void SpawnObstaculoAleatorio()
    {
        // Gera uma coordenada X randômica simulando faixas virtuais na rodovia
        float randomX = Random.Range(-xSpawnRange, xSpawnRange);
        
        // Cria a matriz de posicionamento fixa 1 metro de altura e sempre somando metros à frente do robô
        Vector3 spawnPosition = new Vector3(randomX, 1f, playerTransform.position.z + spawnDistanceAhead);

        // Realiza a clonagem em tempo de execução real na memória do motor de jogos
        Instantiate(prefabPoluicao, spawnPosition, Quaternion.identity);
    }
}

8. Missão 5: Mecanismo de Defesa Ambiental (Disparo de Sementes Purificadoras e Destruição)

Para lidar com bloqueios pesados na pista (barreiras de barris tóxicos), programaremos um sistema de armas ecologicamente correto. O robô poderá gerar projéteis esféricos rápidos pressionando a barra de espaço.

Passo 1: Configuração do Projétil da Semente

Monte uma esfera verde (3D Object > Sphere), adicione a Tag Semente, marque seu colisor como **Is Trigger** e crie o script ProjetilSemente anexado a ela. Salve o arquivo na pasta de prefabs e remova-o da cena de testes.

ProjetilSemente.cs C# / Projectile Calculus 
using UnityEngine;

public class ProjetilSemente : MonoBehaviour
{
    public float projectileSpeed = 25f;  // Velocidade linear de ejeção do projétil
    public float lifeTime = 3f;          // Tempo máximo de vida útil em segundos antes do descarte

    void Start()
    {
        // Destruição agendada de forma nativa para impedir o vazamento crônico de memória na GPU por projéteis perdidos
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        // Desloca o projétil continuamente para frente em alta velocidade
        transform.Translate(Vector3.forward * projectileSpeed * Time.deltaTime);
    }

    void OnTriggerEnter(Collider other)
    {
        // Se colidir contra um objeto sólido rotulado com a tag correspondente da barreira
        if (other.CompareTag("BarreiraToxica"))
        {
            Destroy(other.gameObject);  // Deleta e estraçalha o obstáculo da pista
            Destroy(gameObject);        // Deleta o próprio projétil finalizando o impacto
        }
    }
}

Conexão de Integração no Script Principal

Abra seu código PlayerController.cs e adicione duas propriedades públicas: public GameObject sementePrefab; e public Transform pontoDeDisparo;. Dentro da área da função Update(), cole a condicional de escuta do teclado:

if (Input.GetKeyDown(KeyCode.Space))
{
    Instantiate(sementePrefab, pontoDeDisparo.position, Quaternion.identity);
}

9. Missão 6: O Portal do Santuário Verde (Critério de Vitória e Escala de Tempo)

A última etapa da trilha prática é estabelecer a linha de chegada. Criaremos uma zona mística no extremo fim da pista que detectará a chegada do robô seguro e encerrará os cálculos matemáticos do jogo.

SantuarioPortal.cs C# / Level Complete Logic 
using UnityEngine;

public class SantuarioPortal : MonoBehaviour
{
    void OnTriggerEnter(Collider other)
    {
        // Valida se quem cruzou o portal místico foi de fato o nosso Eco-Robô
        if (other.CompareTag("Player"))
        {
            // Comunica a vitória mudando a variável de estado do GameManager
            GameManager.Instance.isLevelComplete = true;
            Debug.Log("PARABÉNS! O EcoQuest foi concluído com sucesso e a natureza foi restaurada!");
            
            // Paralisa o relógio global interno da engine, congelando movimentos e spawns
            Time.timeScale = 0f;
        }
    }
}

Configuração Final do Mapa: Crie um cilindro gigante posicionado nas coordenadas finais **Z = 45**, mude o nome para Portal_Santuario. Ative a propriedade Is Trigger, atribua a Tag Santuario e anexe o script SantuarioPortal. Seu jogo está mecânica e conceitualmente completo!
